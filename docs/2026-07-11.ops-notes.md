```bash
docker build -t vlandivir-2025 .
docker run -p 3000:3000 vlandivir-2025
```

Production build and deploy go through `.github/workflows/deploy-production.yml`; local docker run with env from `.env` — `run-docker.sh`.

## Prisma

```bash

# verbose output when a migration misbehaves
DEBUG="*prisma*" npx prisma migrate dev
```

## Telegram webhook

Registered automatically on app start, but can be set manually:

```bash
curl -X POST "https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=https://vlandivir.com/telegram-bot"
```

Webhook requires HTTPS — hence the certificate setup below.

## HTTPS certificates (certbot, on the droplet)

Certificates live in `/etc/letsencrypt/live/vlandivir.com/` and are mounted read-only into the container (see the deploy workflow).

Current renewal procedure — standalone challenge needs port 80/443, so stop the container first:

```bash
docker stop vlandivir-2025
certbot certonly --standalone -d vlandivir.com --agree-tos \
  -m vladimir.rybakov@gmail.com --non-interactive \
  --preferred-challenges http --force-renewal
docker start vlandivir-2025
docker logs -f vlandivir-2025
```

As of May 2025 certbot has a scheduled task for auto-renewal. Fallback if standalone doesn't work — manual DNS challenge (requires adding TXT records `_acme-challenge.vlandivir.com` by hand, no auto-renewal):

```bash
sudo certbot certonly --manual --preferred-challenges dns \
  -d vlandivir.com -d www.vlandivir.com --agree-tos \
  --email vladimir.rybakov@gmail.com
```
